## üñºÔ∏è How to Save Diagrams as Images

### Method 1: Right-Click Save
1. Open any `.html` file in your browser
2. Wait for the diagram to fully render
3. Right-click on the diagram
4. Select "Save image as..."
5. Choose your destination folder and save

### Method 2: Browser Screenshot (Recommended for Best Quality)
1. Open the `.html` file in **Google Chrome** or **Microsoft Edge**
2. Press **F12** to open Developer Tools
3. Press **Ctrl + Shift + P** (or Cmd + Shift + P on Mac)
4. Type "screenshot"
5. Select **"Capture full size screenshot"**
6. The image will be saved to your Downloads folder

### Method 3: Third-Party Tools
Use screenshot tools like:
- **Windows:** Snipping Tool, Snip & Sketch (Win + Shift + S)
- **Mac:** Screenshot app (Cmd + Shift + 4)
- **Third-party:** Greenshot, ShareX, Lightshot

## üìê Diagram Types Explained

### Workflow Chart
- **Purpose:** Shows the complete user journey and system processes
- **Use Case:** Understanding application flow, onboarding documentation
- **Format:** Flowchart with decision points and process nodes

### DFD Level 0 (Context Diagram)
- **Purpose:** High-level view of system with external entities
- **Use Case:** System overview, stakeholder presentations
- **Shows:** User, System, External Services (Google OAuth, Email)

### DFD Level 1 (System Overview)
- **Purpose:** Internal processes and data stores
- **Use Case:** Technical documentation, system architecture
- **Shows:** 6 processes (Auth, Profile, Accounts, Transactions, Categories, Reports)

### DFD Level 2 (Transaction Management)
- **Purpose:** Detailed breakdown of transaction processing
- **Use Case:** Development planning, debugging workflows
- **Shows:** 7 sub-processes for transaction CRUD operations

### Class Diagram
- **Purpose:** Object-oriented design and relationships
- **Use Case:** Development reference, database design
- **Shows:** All entities with attributes, methods, and relationships
